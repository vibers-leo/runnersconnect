---
name: 작업 피드백 및 선호도
description: 사용자가 선호하는 작업 방식, 피해야 할 것들
type: feedback
---

- 여러 명령을 한 번에 받아서 한 번에 실행하는 것을 선호 ("다 듣고 한번에 진행")
- 테스트 결과는 직접 확인 후 "자동배포 확인 완료!" 같은 명확한 결론으로 전달
- 작업 완료 후 불필요한 요약이나 설명 반복 없이 핵심만 전달
- 서버 접속 등 실제 실행 가능한 것은 직접 진행하길 원함 (물어보지 말고 실행)
- 인프라 판단(비용, 안정성)에 대해 솔직한 의견 원함

**Why:** 사용자가 여러 프로젝트를 동시 운영하므로 효율적인 소통을 중시함.
